# TattooContact

**TattooContact** est un composant de formulaire de contact spécialisé pour les demandes de tatouage. Il permet aux clients de sélectionner un artiste et de soumettre une demande détaillée avec la possibilité d'ajouter des photos de référence.

## Features

- **Sélection d'artiste** : Menu déroulant (select) pour choisir un artiste dans une liste
- **Formulaire complet** : Collecte toutes les informations nécessaires pour une demande de tatouage
- **Upload de photos avec feature flip** :
  - **Mode Attachment** : Compression et envoi en pièce jointe (limite 500KB total)
  - **Mode Cloud** : Upload sur Cloudinary avec URLs (max 3 photos de 2MB)
- **Validation** : Validation des champs obligatoires et des fichiers uploadés
- **Indicateur de taille** : Barre de progression visuelle pour le mode attachment
- **Gestion des artistes sans email** : Bouton mailto si l'artiste n'a pas d'email configuré
- **Responsive** : Design mobile-first adapté à tous les écrans
- **Accessible** : ARIA labels, hints et messages d'erreur pour une accessibilité optimale
- **CSS Modules** : Styles modernes sans Styled Components

## How It Works

Le composant fonctionne en deux étapes :

### Étape 1 : Sélection de l'artiste
- Affiche un menu déroulant (select) avec la liste des artistes
- L'utilisateur sélectionne un artiste dans la liste
- Une fois sélectionné, l'artiste est automatiquement validé

### Étape 2 : Formulaire ou Mailto
- **Si l'artiste a un email configuré** : Affiche le formulaire complet avec tous les champs
- **Si l'artiste n'a pas d'email** : Affiche un bouton mailto vers l'email général

### Champs du formulaire complet
- Email (obligatoire)
- Nom complet (obligatoire)
- Téléphone (obligatoire)
- Objet de la demande (obligatoire)
- Description du projet (obligatoire, textarea)
- Taille du tatouage en cm (obligatoire)
- Zone du tatouage (obligatoire)
- Disponibilités (obligatoire)
- Budget estimé (optionnel)
- Photos de référence (optionnel, max 5 fichiers de 2MB)

## Usage

### Exemple basique avec plusieurs artistes

```jsx
<TattooContact
  artists={[
    { artistName: "John Doe", mail: "john.doe@example.com" },
    { artistName: "Jane Smith", mail: "jane.smith@example.com" },
    { artistName: "Mike Johnson", mail: "mike.johnson@example.com" }
  ]}
  serviceId="service_xxxxx"
  templateId="template_xxxxx"
  replyTo="contact@example.com"
/>
```

### Avec un artiste sans email

```jsx
<TattooContact
  artists={[
    { artistName: "John Doe", mail: "john.doe@example.com" },
    { artistName: "Jane Smith", mail: null } // Pas d'email = bouton mailto
  ]}
  serviceId="service_xxxxx"
  templateId="template_xxxxx"
  replyTo="contact@example.com"
/>
```

```json
            "width": 1920,
            "height": 1080,
            "crop": "limit",
            "quality": "auto:good"
          }

      }
    }
  }

```

### Mode Attachment (avec EmailJS)

### Configuration sans features (par défaut = cloud)

Si `features` n'est pas spécifié, le mode cloud est utilisé par défaut avec la configuration minimale.

```json
{
  "type": "contact",
  "id": "tattooContact",
  "datas": {
    "artists": [
      {
        "artistName": "Artist Name",
        "mail": "artist@example.com"
      }
    ]
  }
}
```

### Configuration des artistes

Chaque artiste doit avoir :
- **artistName** (string, obligatoire) : Le nom complet de l'artiste
- **mail** (string | null, obligatoire) : L'adresse email de l'artiste ou `null` si pas d'email

**Important** :
- Si `mail` est `null`, un bouton mailto vers l'email général sera affiché au lieu du formulaire
- Les artistes apparaissent dans l'ordre du tableau

### Configuration des features - Images Display

Le composant supporte deux modes d'upload d'images via le système de feature flip :

#### Mode Cloud (Cloudinary) - Recommandé

**Avantages** :
- ✅ Jusqu'à 3 photos de 2MB chacune (après réduction légère)
- ✅ URLs permanentes
- ✅ CDN global rapide
- ✅ Transformations automatiques
- ✅ Aperçu thumbnail dans le formulaire

**Configuration** :

```json
{
  "features": {
    "imagesDisplay": {
      "type": "cloud",
      "cloud": {
        "cloudName": "your-cloudinary-account",
        "uploadPreset": "unsigned_preset",
        "folder": "tattoo-references",
        "maxPhotos": 3,
        "transformation": {
          "width": 1920,
          "height": 1080,
          "crop": "limit",
          "quality": "auto:good"
        }
      }
    }
  }
}
```

**Champs** :
- `cloudName` (string, requis) : Nom de votre compte Cloudinary
- `uploadPreset` (string, requis) : Nom du preset unsigned (voir setup ci-dessous)
- `folder` (string, optionnel) : Dossier de destination dans Cloudinary
- `maxPhotos` (number, défaut: 3) : Nombre max de photos
- `transformation` (object, optionnel) : Transformations automatiques
  - `width` : Largeur max en pixels
  - `height` : Hauteur max en pixels
  - `crop` : Mode de crop (`"limit"`, `"fit"`, `"fill"`)
  - `quality` : Qualité (`"auto:good"`, `"auto:best"`, `1-100`)

**Setup Cloudinary** :

1. Créer un compte gratuit : https://cloudinary.com/users/register/free
2. Récupérer le **Cloud Name** (Dashboard → Account Details)
3. Créer un **Upload Preset** :
   - Settings → Upload → Upload presets
   - Add upload preset
   - **Signing Mode** : Unsigned ⚠️
   - **Folder** : `tattoo-references`
   - **Max file size** : 10MB
   - **Allowed formats** : jpg, png, webp
   - Copier le **Preset name**

**Template EmailJS pour mode Cloud** :

```html
{{#if photoUrls}}
<h3>Photos de référence</h3>
{{#each photoUrls}}
<div>
  <a href="{{this}}">
    <img src="{{this}}" alt="Photo" style="max-width: 400px;" />
  </a>
</div>
{{/each}}
{{/if}}
```

#### Mode Attachment (EmailJS) - Legacy

**Avantages** :
- ✅ Simple à configurer
- ✅ Pas de service externe
- ✅ Gratuit

**Limitations** :
- ❌ Limite à 2 photos maximum
- ❌ 500KB total (limite EmailJS)
- ❌ Qualité réduite par compression
- ❌ Pas d'URLs permanentes

**Configuration** :

```json
{
  "features": {
    "imagesDisplay": {
      "type": "attachment",
      "attachment": {
        "maxTotalSizeKB": 500,
        "maxPhotos": 2,
        "targetSizePerPhotoKB": 240,
        "maxResolution": 1400,
        "compressionQuality": 0.85
      }
    }
  }
}
```

**Champs** :
- `maxTotalSizeKB` (number, défaut: 500) : Taille totale max (limite EmailJS)
- `maxPhotos` (number, défaut: 2) : Nombre max de photos
- `targetSizePerPhotoKB` (number, défaut: 240) : Taille cible par photo après compression
- `maxResolution` (number, défaut: 1400) : Résolution max (largeur/hauteur)
- `compressionQuality` (number, défaut: 0.85) : Qualité compression (0-1)

**Template EmailJS pour mode Attachment** :

```html
<p>{{photoCount}} photo(s) de référence en pièce jointe</p>
```

Les images arrivent automatiquement en pièces jointes de l'email.

### Configuration EmailJS

Le service EmailJS doit être configuré dans le Redux store via `getClient().contact` avec :

- **mailServiceId** : Votre ID de service EmailJS
- **mailTemplate** : Votre ID de template EmailJS

Le template doit accepter ces variables :
- `artistName` : Nom de l'artiste sélectionné
- `artistMail` : Email de l'artiste
- `email` : Email du client
- `name` : Nom complet du client
- `phone` : Téléphone du client
- `object` : Objet de la demande
- `description` : Description du projet de tatouage
- `tattooSize` : Taille du tatouage
- `tattooZone` : Zone du tatouage
- `availability` : Disponibilités du client
- `budget` : Budget estimé (optionnel)
- `photoCount` : Nombre de photos
- **Mode attachment** : `photos` (pièces jointes)
- **Mode cloud** : `photoUrls` (array de URLs), `photoUrl_0`, `photoUrl_1`, etc.

### Exemple de template EmailJS

```
Nouvelle demande de tatouage

Artiste sélectionné : {{artistName}} ({{artistMail}})

--- Informations client ---
Nom : {{name}}
Email : {{email}}
Téléphone : {{phone}}

--- Détails du projet ---
Objet : {{object}}
Description : {{description}}

Taille : {{tattooSize}}
Zone : {{tattooZone}}
Disponibilités : {{availability}}
Budget : {{budget}}

--- Photos ---
{{photoNames}}
```

## Best Practices

### Nombres d'artistes recommandés
- **1-3 artistes** : Design optimal sur tous les écrans
- **4-6 artistes** : Bon rendu sur tablette et desktop
- **7+ artistes** : Considérer une pagination ou un système de filtres

### Configuration EmailJS
1. Créez un compte sur [EmailJS](https://www.emailjs.com/)
2. Créez un service email (Gmail, Outlook, etc.)
3. Créez un template avec toutes les variables listées ci-dessus
4. Ajoutez votre Public Key dans le composant (`emailjs.init()`)
5. Testez avec la version Storybook avant de déployer

### Gestion des photos

**Mode Cloud (Cloudinary)** :
- Les images sont uploadées sur Cloudinary avant l'envoi de l'email
- Réduction légère à 1920x1080 max avec qualité auto:good
- Maximum 3 photos de 2MB chacune après réduction
- URLs permanentes transmises dans l'email
- Aperçu thumbnail dans le formulaire

**Mode Attachment (EmailJS)** :
- Les images sont compressées côté client avec browser-image-compression
- Maximum 2 photos pour rester sous la limite de 500KB
- Taille cible de 240KB par photo
- Envoyées en pièces jointes de l'email
- Indicateur visuel de taille avec barre de progression

### Accessibilité
- Tous les champs ont des labels visibles
- Les hints fournissent des exemples concrets
- Les champs obligatoires sont marqués avec `*` et `aria-required`
- Les erreurs de photos sont annoncées avec `role="alert"` et `aria-live="polite"`
- Support complet du clavier pour la navigation
